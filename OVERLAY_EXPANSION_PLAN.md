# iRon-with-ai: 오버레이 확장 계획 (iOverlay 기반)

이 문서는 `iRon-with-ai` 프로젝트의 기능 확장 계획을 정리합니다. 본격적으로 AI 기능을 접목하기 전에, 현재 심레이싱 유저들에게 가장 인기 있는 **iOverlay**의 핵심 기능들을 벤치마킹하여 iRon의 기본 오버레이 성능을 대폭 끌어올리는 것을 목표로 합니다.

## 1. 현재 상태 비교

| 기능 분류 | iRon (현재) | iOverlay (참조 타겟) |
| :--- | :--- | :--- |
| **순위표 (Standings)** | 기본 (이름, SR, iRating, 피트 횟수, 친구 강조) | 고도화된 커스텀 (21+ 컬럼: 실시간 iRating 증감 예측, 차량 브랜드, 랩타임 간격, 피트 상태 등) |
| **상대 거리 (Relative)** | 기본 + 선택형 미니맵 | 기본 + 정밀한 트랙 복귀 및 트래픽 간격 계산 |
| **트랙 맵 (Track Map)** | 기본 미니맵 (절대/상대 위치) | 2D 트랙 맵 + **일자형 맵(Flat Map: 전체 트랙의 트래픽 밀도를 한눈에 파악)** |
| **연료 계산기 (Fuel)** | 예상 남은 랩, 필요 연료, 랩당 소모량(최근 4랩 평균) | 정밀 트래킹 + **팀 세션 연료 데이터 실시간 공유** |
| **입력 그래프 (Inputs)** | 실시간 움직이는 그래프 (스로틀, 브레이크, 스티어링) | 실시간 그래프 + 트레일 브레이킹/스로틀 전개 히스토리 분석 기능 |
| **주변 인지 / 레이더** | ❌ 없음 | **시각적 스포터 (좌/우 차량 알림), 사고 발생 경고, 근접 레이더 (차량 사각지대 시각화)** |
| **멀티클래스 지원** | ❌ 없음 | **상위 클래스 차량 접근 경고 (블루 플래그 대응 지원)** |
| **피트 도우미 (Pit)** | ❌ 없음 | **피트 헬퍼 (진입/진출 타이밍, 정차 시간 타이머 표시)** |
| **스트리밍 / 편의성** | 가림막 (빈 사각형), JSON 및 Alt+J로 설정 | 트위치 채팅 연동, 차고(세팅) 화면 자동 가림막, **OBS 브라우저 소스 지원, 세션별 레이아웃 자동 전환** |

---

## 2. 권장 구현 기능 목록

`iRon-with-ai`를 현대적인 오버레이 수준으로 끌어올리기 위해 다음 기능들을 우선적으로 개발합니다.

### 우선순위 1: 안전 및 상황 인지 도구 (가장 수요가 많은 기능)
*   **시각적 스포터 / 레이더:** ACC(아세토 코르사 컴페티치오네)와 유사한 근접 레이더를 구현하여 사각지대의 차량을 시각적으로 표시합니다. 직관적인 좌/우 방향 지시기를 추가합니다.
*   **멀티클래스 트래픽 경고:** 상위 클래스의 빠른 차량이 뒤에서 특정 시간(예: 3초) 이내로 접근할 때 화면에 띄우는 경고 UI를 구현합니다.
*   **황기(Yellow Flag) / 사고 알림:** iRacing 텔레메트리의 깃발 데이터를 읽어 전방 사고 발생 시 팝업 형태의 시각적 경고를 띄웁니다.

### 우선순위 2: 텔레메트리 및 대시보드 고도화
*   **일자형 맵 (Flat Map / Ribbon Map):** 트랙을 일직선으로 펼쳐서 보여주는 맵을 구현합니다. 내구 레이스나 멀티클래스 경기에서 앞뒤 트래픽 팩이 어디에 뭉쳐있는지 파악하는 데 필수적입니다.
*   **순위표 확장:** 레이스 중 예상 iRating 증감 수치(+/-)를 실시간으로 보여주고, 차량 브랜드 로고 등을 텍스트로 추가합니다.
*   **델타 바 (Delta Bar):** 베스트 랩 또는 세션 최적 랩 대비 현재 랩타임 차이를 빨간색/초록색 바 형태로 직관적으로 보여줍니다.
*   **피트 헬퍼 (Pit Helper):** 피트레인 진입 시 자동으로 나타나며, 피트 속도 제한, 피트 박스까지의 카운트다운, 정차 시간 타이머를 표시하는 전용 오버레이입니다.

### 우선순위 3: 편의성(QoL) 및 스트리밍 지원
*   **세션별 레이아웃 저장:** 연습(Practice), 예선(Qualy), 결승(Race) 세션에 따라 오버레이의 위치나 표시 여부가 자동으로 전환되도록 설정 기능을 추가합니다.
*   **OBS 연동 (선택사항):** 화면 캡처 시 오버레이가 깔끔하게 방송되도록, 궁극적으로는 렌더링 방식을 HTML/JS 기반(웹 브라우저 소스)으로 확장하는 것을 고려합니다. (현재의 DirectX 방식 구조 개선 필요)
