# iRon-with-ai 개발 태스크 (Task List)

## 📌 Phase 1: 기반 시스템 점검 및 구조 개선
- [x] **iRacing 텔레메트리 데이터 분석:** 신규 기능(주변 차량 좌표, 플래그 상태, 예상 iRating 등)에 필요한 데이터가 현재 `iracing.cpp`를 통해 수신 가능한지 iRacing SDK 문서 확인.
- [x] **UI 렌더링 프레임워크 검토:** 현재 C++ / DirectX 기반의 UI가 레이더나 일자형 맵 같은 복잡한 도형을 그리는 데 적합한지 평가. (필요시 ImGui 등 경량 GUI 라이브러리 도입 검토)

## 🔴 Phase 2: 안전 및 상황 인지 도구 (우선순위 1)
- [x] **시각적 스포터 (Spotter):** 
  - [x] 좌/우 차량 근접 시 화면 좌우측에 시각적 경고(색상 변화 등) 띄우기 로직 구현.
- [ ] **근접 레이더 (Radar):**
  - [ ] 내 차량을 중심으로 반경 N미터 이내의 타 차량 상대 좌표 계산.
  - [ ] 계산된 좌표를 기반으로 하단에 탑다운 뷰 형식의 레이더 그래픽 렌더링.
- [ ] **멀티클래스 트래픽 경고:**
  - [ ] 상위 클래스 판별 로직 추가.
  - [ ] 타겟 차량과 내 차량 간의 예상 조우 시간(Delta Time) 계산.
  - [ ] N초 이내 접근 시 블루 플래그 형태의 경고 UI 출력.
- [ ] **사고/황기 알림:**
  - [ ] iRacing 플래그 텔레메트리 연동.
  - [ ] 황기 발생 시 오버레이 상단에 경고 배너 출력.

## 🟡 Phase 3: 텔레메트리 및 대시보드 고도화 (우선순위 2)
- [ ] **일자형 맵 (Flat Map):**
  - [ ] 전체 트랙 길이를 0~100% 비율로 정규화.
  - [ ] 모든 차량의 트랙 위 위치를 일직선 바(Bar) 형태로 매핑하여 렌더링.
- [ ] **순위표(Standings) 확장:**
  - [ ] 실시간 예상 iRating 증감 계산 공식 적용 및 UI 컬럼 추가.
  - [ ] 차량 엠블럼 또는 제조사 텍스트 컬럼 추가.
- [ ] **델타 바 (Delta Bar):**
  - [ ] 베스트 랩 / 옵티멀 랩 데이터 수신.
  - [ ] 실시간 델타 차이를 가로형 게이지(빨강/초록)로 렌더링.
- [ ] **피트 헬퍼 (Pit Helper):**
  - [ ] 피트레인 진입 상태 감지 이벤트 추가.
  - [ ] 진입 시 전용 오버레이 활성화 (속도 제한, 정차 타이머 표시).

## 🟢 Phase 4: 편의성(QoL) 및 스트리밍 지원 (우선순위 3)
- [ ] **세션별 레이아웃 전환:**
  - [ ] 현재 세션 상태(Practice, Qualify, Race) 감지.
  - [ ] `config.json`에 세션별 오버레이 위치/표시 여부 프로필 저장 및 자동 로드 기능 구현.
- [ ] **OBS 브라우저 소스 지원 (선택/장기 과제):**
  - [ ] C++ 백엔드에서 텔레메트리 데이터를 웹소켓(WebSocket)으로 쏘는 로컬 서버 구축.
  - [ ] HTML/JS/CSS 프론트엔드를 구축하여 브라우저에서 오버레이 렌더링하도록 전환.
